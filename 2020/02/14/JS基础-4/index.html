    BOM

    BOM —（Browser Object Model）

    window

    BOM的最顶层对象

    open(); 打开

    close(); 关闭

    文档宽度高度

    innerWidth 返回窗口的文档显示区的宽度

    innerHeight 返回窗口的文档显示区的高度

    使用 innerWidth 和 innerHeight 属性获取去除工具条与滚动条的窗口高度与宽度

    outerWidth 属性设置或返回窗口的外部宽度，包括所有的界面元素（如工具栏/滚动条）

    outerHeight 属性设置或返回一个窗口的外部高度，包括所有的界面元素（如工具栏/滚动条）

    使用 outerWidth 和 outerHeight 属性获取加上工具条与滚动条窗口的宽度与高度

    到屏幕左上角位置

    screenLeft，screenTop

    screenX，screenY

    window.location本地

    location.reload(); 重新载入

    ​ F5 调用缓存

    ​ reload 请求服务时，没有在http标签头部消息中有缓存标记，意味着不会调用缓存数据

    location.href="http://www.163.com";
//不会在新窗口打开，有历史记录
    location.assign="http://www.163.com";
//不会在新窗口打开，有历史记录
    location.replace="http://www.163.com";
//不会在新窗口打开，没有历史记录，不能回退

    location.hash

    浏览器地址栏中的”#”后面的内容，常用于路由跳转，超链接的锚点标记

    location.search();

    浏览器地址栏中的”?”后面的内容，表单的GIT数据提交或者页面的传参

    location.hostname; 域 IP地址不带端口号

    location.port; 端口号

    location.protocol; 协议 例如：http、https

    window.history 历史

    锚点标记可以设置在普通标签里，也可以设置在a标签的内容里

    当前页面的锚点，不会跳转页面，不会重新执行当前页面的script

    历史记录会记录当前页面中所有操作过的内容，包括文本框中的内容（除了密码），都会保留下来

    history.back(); 后退一步

    history.length(); 历史记录数

    history.forward(); 前进一步

    history.go(0); 刷新页面，会重新执行script标签内容

    history.go(1); 同forward

    history.go(-1); 同back

    window.screen屏幕信息

    屏幕不含任务栏的高度与宽度

    ​ screen.availWidth

    ​ screen.availHeight

    带屏幕下方任务栏的高度

    ​ screen.width

    ​ screen.height

    window.navigator导航信息

    navigator.userAgent 浏览器的信息

    navigator.appName 返回浏览器的名称

    navigator.appversion 版本号

    navigator.platform 运行平台，判断手机系统

    DOM

    DOM基础

    外链样式
    从外部加载css样式，仅加载进来 
    <script src="a.js"></script>
//加载进来之后执行该script标签内容。同步顺序，或导致加载拥塞
    <script src="a.js" async></script>
//异步过程 加载的过程中会执行下一个标签
    <script src="a.js" defer></script>
//页面全部加载完成之后在执行该script标签
    基本标签

    window 是BOM的根节点

    document 是DOM的根节点

    document.documentElement    //HTML标签
document.head    //head标签
document.body    //body标签
document.title    //title标签
document.domain    //域名
    页面加载执行原理

    生成DOM树。从根节点开始，按照标签的顺序逐个生成

    生成CSS树。外链css、style中的css、行内样式统一生成一个css树

    合并生成页面的渲染树

    如果需要获取已经完成渲染后的DOM样式时

    首先需要将渲染树拆分为DOM树和CSS树两部分

    然后再从css中获取计算后的样式，实际核算中属于同一个拆分过程

    元素标签的检查与判断

    console.log(div); 通过标签名获取元素并打印出HTMLCollection[div]

    console.dir(div); 打印出div对象

    var div0=document.getElementsByClassName("div0");
div0.constructor===HTMLDivElement; //判断元素是否是div标签
ul.constructor===HTMLUListElement; //ul
a.constructor===HTMLAchorElement; //超链接

    HTMLDivEelement–>HTMLElement–>Element–>Node–>EventTarget–>Object

    DOM节点

    document 是文档节点 9

    div a span 都是元素节点 1

    标签属性 是属性节点

    标签中的文本 是文本节点 3

    注释 是注释节点 8

    nodeName

    ​ 文本 #text

    ​ 注释 #comment

    ​ 元素 标签名（大写）

    nodeValue

    ​ 注释 注释内容

    ​ 文本 文本内容

    ​ 元素 null

    节点分类

    均以document.body开头，表示body下的

    ​ children //子元素列表

    ​ childNodes //子节点列表

    ​ parentNode //父节点

    ​ parentElement //父元素

    ​ firstChild //第一个子节点

    ​ firstElementChild //第一个子元素

    ​ lastChild //最后一个子节点

    ​ lastElementChild //最后一个子元素

    ​ previousSibling //上一个兄弟节点

    ​ previousElementSibling //上一个兄弟元素

    ​ nextSibling //下一个兄弟节点

    ​ nextElementSibling //下一个兄弟元素

    注意：节点包括元素，元素仅指标签

    获取DOM元素的方法
    document.getElementById("id");
