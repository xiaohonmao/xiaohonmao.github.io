    文档宽度高度

    innerWidth 返回窗口的文档显示区的宽度

    innerHeight 返回窗口的文档显示区的高度

    使用 innerWidth 和 innerHeight 属性获取去除工具条与滚动条的窗口高度与宽度

    outerWidth 属性设置或返回窗口的外部宽度，包括所有的界面元素（如工具栏/滚动条）

    outerHeight 属性设置或返回一个窗口的外部高度，包括所有的界面元素（如工具栏/滚动条）

    使用 outerWidth 和 outerHeight 属性获取加上工具条与滚动条窗口的宽度与高度

    抽象

    function ce(type,style,parent,textContent,prop){
    var elem=document.createElement(type);
    for(var p1 in style){
        elem.style[p1]=style[p1];
    }
    for(var p2 in prop){
        elem[p2]=prop[p2];
    }
    if(textContent) elem.textContent=textContent;
    if(parent) parent.appendChild(elem);
    return elem;
}

    DOM位置*

    div

    ​ 坐标点就是左上顶点

    div.clientLeft，div.clientTop

    ​ 从padding以外到坐标点位置的距离（边线粗细）

    ​ clientLeft–>左边线粗细

    ​ clientTop–>顶边线粗细

    ​

    div.offsetLeft，div.offsetTop

    ​ 如果没有使用定位，并且其向上所有父元素都没有定位的时候，offsetLeft是距离html左端距离，offsetTop是距离html顶部的距离

    ​ 如果该元素使用了定位，offsetLeft和offsetTop就是该元素定位后left和top值定位，有left，top样式

    ​ offsetLeft–>left，offsetTop–>top

    ​ 如果没有设置left和top值，父元素设置了定位

    ​ offsetLeft–>当前元素左边线到父元素的左边线位置

    ​ offsetTop–>当前元素顶边线到父元素顶边线的位置

    ​ 如果没有设置left和top值，父元素没有定位

    ​ 向上查找父元素及以上是否有定位，直到html标签为止

    ​ offsetLeft–>当前元素到该定位的父元素的左边线位置

    ​ offsetTop–>当前元素到该定位的父元素的顶边线位置

    div.scrollLeft，div.scrollTop

    ​ 唯一可以被设置值的

    var div=document.querySelector(".div2");

function scrollHeadler(e){

//如果直接在这里设置时，div的样式css树还没有渲染到渲染树中
//当整个页面全部渲染完成后设置才有作用
//直接给值，不需要带单位
div.scrollTop=100; 
document.addEventListener("click",clickHandler);

    div.scrollTop=100;
    div.scrollLeft=100;
}

    事件基础

    事件需要先侦听后抛发

    //侦听事件
document.addEventListener("chilma",eventHandler);
function eventHandler(e){
    console.log("aaa");
}
//抛发事件
var evt=new Event("chilma");
document.dispatchEvent(evt);

    案例

    var bn=document.querySelector("button");
bn.addEventListener("click",clickHandler);
//事件对象
var evt=new MouseEvent("click");
evt.sum=100;
//针对侦听对象抛发事件，将事件对象抛发给侦听者
bn.dispatchEvent(evt);
//e就是抛发的对象
function clickHandler(e){
    console.log(e.sum);
}

    事件原理

    不是任何对象都可以侦听和抛发事件

    所有的EventTarget这个类别可以完成事件的侦听和抛发

    HTMLDivElement–>HTMLElement–>Element–>Node–>EventTarget–>Object

    所有的节点、标签、元素都可以事件侦听和抛发

    事件三阶段（事件原理）

    捕获阶段 外 –> 内

    目标阶段

    冒泡阶段 内 –> 外

    案例

    var div0=document.querySelector(".div0");
div0.addEventListener("click",clickHandler(),true);
//第三个参数 是否使用捕获阶段触发，默认值是false（默认是冒泡阶段触发）
function clickHandler(e){
    e.stopPropagation();    //阻止冒泡
    //e.cancalBubble=true;    //IE8以下的兼容
    console.log(this.e.currentTarget,e.target,e.srcElement);
}

    e.currentTarget 就是被侦听的对象，也是事件函数中this默认值

    e.target 就是目标对象，实际侦听到的对象

    e.srcElement 等同于e.target也是目标对象

    事件委托

    将需要侦听的对象委托给它的父容器

    如果需要侦听多个元素时，只需要委托给父元素侦听就可以了

    这样可以大大节省侦听事件的创建，以达到减少内存的占有
