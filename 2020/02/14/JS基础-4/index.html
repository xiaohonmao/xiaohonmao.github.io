    DOM位置*

    div

    ​ 坐标点就是左上顶点

    div.clientLeft，div.clientTop

    ​ 从padding以外到坐标点位置的距离（边线粗细）

    ​ clientLeft–>左边线粗细

    ​ clientTop–>顶边线粗细

    ​

    div.offsetLeft，div.offsetTop

    ​ 如果没有使用定位，并且其向上所有父元素都没有定位的时候，offsetLeft是距离html左端距离，offsetTop是距离html顶部的距离

    ​ 如果该元素使用了定位，offsetLeft和offsetTop就是该元素定位后left和top值定位，有left，top样式

    ​ offsetLeft–>left，offsetTop–>top

    ​ 如果没有设置left和top值，父元素设置了定位

    ​ offsetLeft–>当前元素左边线到父元素的左边线位置

    ​ offsetTop–>当前元素顶边线到父元素顶边线的位置

    ​ 如果没有设置left和top值，父元素没有定位

    ​ 向上查找父元素及以上是否有定位，直到html标签为止

    ​ offsetLeft–>当前元素到该定位的父元素的左边线位置

    ​ offsetTop–>当前元素到该定位的父元素的顶边线位置

    事件原理

    不是任何对象都可以侦听和抛发事件

    所有的EventTarget这个类别可以完成事件的侦听和抛发

    HTMLDivElement–>HTMLElement–>Element–>Node–>EventTarget–>Object

    所有的节点、标签、元素都可以事件侦听和抛发

    事件三阶段（事件原理）

    捕获阶段 外 –> 内

    目标阶段

    冒泡阶段 内 –> 外

    案例

    var div0=document.querySelector(".div0");
div0.addEventListener("click",clickHandler(),true);
//第三个参数 是否使用捕获阶段触发，默认值是false（默认是冒泡阶段触发）
function clickHandler(e){
    e.stopPropagation();    //阻止冒泡
    //e.cancalBubble=true;    //IE8以下的兼容
    console.log(this.e.currentTarget,e.target,e.srcElement);
}

    e.currentTarget 就是被侦听的对象，也是事件函数中this默认值

    e.target 就是目标对象，实际侦听到的对象

    e.srcElement 等同于e.target也是目标对象

    事件委托

    将需要侦听的对象委托给它的父容器

    如果需要侦听多个元素时，只需要委托给父元素侦听就可以了

    这样可以大大节省侦听事件的创建，以达到减少内存的占有

    注意：事件函数是不能带入参数的，并且也不能使用return返回值

    节流*

    var input=document.querySelector("input");
input.addEventListener("input",inputHandler);
var ids=0;
function inputHandler(e){
    if(ids) return;
    //setTimeout返回的值一定大于0
    ids=setTimeout(function(){
        clearTimeout(ids);
        ids=0; 
        //这里清除setTimeout以后，还要将ids设为0，这样就可以再次进入时不再跳出
        console.log(input.value);
    },500);
    console.log(ids);
}

    正则表达式

    字面量正则表达式

    var reg=/正则内容/修饰符；

    构造函数正则表达式

    var reg=new RegExp(“正则内容”,”修饰符”);

    修饰符可以写也可以不写

    ​ i 正则表达式匹配时，不区分大小写

    ​ g 全局匹配，可以一直搜索到尾部

    ​ m 多行匹配

    处理正则表达式有两种处理方式，一种是正则表达式的方法，一种是字符串的方法

    正则表达式方法

    var reg=/a/i;
    //判断字符串是否满足正则表达式 test
var bool=reg.test("Abcd");
    //exec 类似于match可以查找到该字符串中满足正则表达式位置，以及群组，但是g无效
console.log(reg.exec("abcda"));

    字符串方法

    字符串可以处理正则表达式的方法

    str.search()

    查找满足正则表达式内容所在下标（第几位）

    var str="Abcdefagd";
//只能搜索到第一个满足条件的
//可以搜索区分大小写的
console.log(str.search(/a/i));

//可以查找不同内容，[ad]表示字母a或者d
console.log(str.search(/[ad]/i));
//search比indexOf更强大，因为可以搜索到变化的字符或者符合规格的字符，indexOf只能搜索到固定的字符
//search找到第一个满足正则的位置

    str.match()

    查找所有满足正则表达式内容的部分产生一个数组

    var str="Abcdefagd";
//match是找到符合正则表达式的字符，这个字符是什么
console.log(str.match(/a/gi));
console.log(str.match(/[ad]/gi));

    str.replace()

    字符串中替换正则表达式内容，可以替换多个

    var str="Abcdefagd";
var s=str.replace("a","z");
//使用正则后，尤其增加了i和g，不区分大小写和全局匹配，可以替换所有的满足正则的字符
var s1=str.replace(/a/ig,"z");
var s2=str.replace(/[ad]/ig,"z");
//将不同的字符所有字符替换成同一个字符
var s3=str.replace(/[ad]/ig,function(item,index){
    //item 匹配正则查找到的字符
    //index 该字符所在下标
})

    按照正则表达式进行切割

    var str="a-b&c-d&e-f";
console.log(str.split(","));
//使用正则表达式可以切割成不同的内容
console.log(str.split(/[&-]/g));

    包含通配符和转义字符

    .通配符，代替任何一个字符

    转移字符使用\

    元字符中括号[]表示中括号中任意一个字符，中括号中重复没有意义

str.match(/[A-Z]/g); //A-Z

    //"1234512345 12345"删除123以外的字符

    重复

    贪婪匹配*

    重复次数的范围

    {3,6} {最少匹配，最多匹配} 优先匹配最多

    少于最少匹配数量则不匹配

    任何字符{0}都表示空字符

    console.log("abab".match(/a{0}/g));
//查找空字符

    {0,1} 有或者没有都满足条件

    console.log("hours".match(/hours{0,1}/));

    {最小，} 最小到无限匹配

    "aaa".match(/a{1,}/);    //至少一次重复，最多无限
"aaaa".match(/a{0,1}/);    //有或者没有都行，最多无限

    {0,1} ?

    {0,} *

    {1,} +

    非贪婪匹配

    ？出现在重复次数后面

    非贪婪匹配，一般用在两个字符之间匹配最少内容

    var str="<div><a href='#'></a></div>"
console.log(str.match(/<.*?>/g));

//输出结果
/*["<div>", "<a href='#'>", "</a>", "</div>"]
    0: "<div>"
    1: "<a href='#'>"
    2: "</a>"
    3: "</div>"
*/

    | 或者 选择匹配

    console.log("catdog".match(/cat|dog/g));    // ["cat", "dog"]
// |空字符串| 
console.log("catdog".match(/cat| |dog/g));    //["cat", "", "", "", ""]

    群组、起始和结束

    起始和结束

    ^ $ 只能出现在最前和最后 /^ $/

    一般用于判断是否满足条件

    案例

    var str="    abcd  abcd   ";
console.log(str.match(/^(\s*)(.*?)(\s*)$/)[2]); //abcd  abcd
//起始位置后跟的内容容易被找到，但是结束位置如果被前面包含时
//注意前面的内容使用非贪婪匹配可以解决结束位置的查找

    案例

    //trim 原理
function trim(str){
    return str.match(/^(\s*)(.*?)(\s*)$/)[2];
}
console.log(trim("    abcd  abc a  ")); //abcd  abc a

    (? = n) 作为条件判断某个内容后紧随其后的是n

    (? ! n) 作为条件判断某个内容后紧随其后的不是n

    （）群组

    使用群组时，尽量不要使用全局匹配，如果使用全局匹配match(exec)时不能获得群组内容
